                     5. DUPLICATE API ANALYSIS
================================================================================

NO DUPLICATE APIs FOUND.

All APIs have distinct purposes. Some apparent overlaps are intentional:

1. Subject Events vs Study Events:
   - /api/events/subject/:subjectId - Events for a specific subject
   - /api/events/study/:studyId - Event definitions for a study
   - /api/subjects/:id/events - Same as subject events (convenience)
   These serve different use cases and should remain separate.

2. Dashboard Stats vs Individual Stats:
   - Dashboard provides aggregated views for the UI
   - Individual service endpoints provide detailed data
   - Both are needed for different components

3. Query Stats:
   - /api/dashboard/queries - Dashboard summary
   - /api/queries/stats - Detailed query statistics
   - Both serve UI dashboard and query management views
